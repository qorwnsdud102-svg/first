# wiki/ — 정리된 위키 (제2의 뇌 본체)

이 폴더가 진짜 "제2의 뇌"다. 모든 페이지는 **하나의 엔티티** (개념·사람·책·사건·질문)에 집중한다.
"잡다한 메모"는 wiki에 들어오면 안 된다 — 그건 raw거나, 아직 정리 안 된 초안.

## 진입점

- [[index]] — 위키 전체의 텍스트 진입 페이지. 자주 가는 곳·미해결 질문·분류별 링크.
- [[log]] — 일자별 활동 로그. Ingest·Lint·구조 변경은 모두 한 줄씩 여기.

## 하위 폴더 — 어디에 새 페이지를 두는가

새 페이지는 다음 4곳 중 하나에 들어간다. `type:` frontmatter와 폴더는 보통 일치한다.

- `sources/` — raw 파일 1개당 페이지 1개. 출처·요지·어느 엔티티에 영향을 줬는지 추적.
- `concepts/` — 추상 개념. (예: [[어텐션]], [[RAG]], [[제2의 뇌]])  → `type: 개념`
- `entities/` — named: 사람·책·조직·구체 작품. (예: [[Karpathy]], [[Attention Is All You Need]])  → `type: 사람` · `책` 등
- `syntheses/` — 여러 페이지를 가로지르는 내 종합·비교·가설. wiki의 부가가치.  → `type: 질문` 또는 신규 `종합` 타입

## 페이지 형식 (템플릿)

````markdown
---
type: 개념 | 사람 | 책 | 사건 | 질문
aliases: [별칭1, 별칭2]
status: stub | growing | stable
sources: [raw/파일명.md, raw/다른파일.md]
updated: 2026-05-09
---

# 엔티티 이름

## 한 줄 정의

- [[또 다른 엔티티]] — ...
````

## 새 페이지 만들기 전에 (중복 방지)

- 같거나 유사한 엔티티가 이미 있는가? Obsidian 전역 검색·그래프뷰로 먼저 확인.
- 있으면 합친다. 별칭이 다르면 frontmatter `aliases:`에 추가하고 페이지는 하나로 유지.
- "비슷한데 다르다"는 판단이 들면, 두 페이지를 만들고 `[[A]] vs [[B]]` 같은 비교 페이지를 따로 만든다.

## Lint — 주기적으로 확인

- [ ] 출처 0개인 페이지 → `status: stub`로 강등 또는 삭제.
- [ ] orphan (어디서도 링크되지 않은 페이지) → 의도적인지 확인, 아니면 연결.
- [ ] 모순되는 주장 → 삭제하지 말고 페이지 안에 `## 미해결 충돌` 섹션으로 보존.
- [ ] 14일 이상 `stub` → 합치거나 raw로 강등.
- [ ] `updated`가 오래된 핵심 페이지 → 최신 raw로 재검토.

## Query — wiki를 읽어 답할 때

- 1차 출처는 wiki, raw는 보조.
- 답이 wiki에 없으면 **추측 금지**. "wiki에 없음 — raw `XYZ`에는 있을 수 있음" 형식으로 보고.
- 답이 여러 페이지에 걸쳐 있으면 출처 페이지 모두 인용.
