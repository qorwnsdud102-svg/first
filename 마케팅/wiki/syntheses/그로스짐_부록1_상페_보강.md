# (부록 1) 상세페이지 특성 및 설계

## 한 줄 요지

상세페이지를 **"생명체"**로 취급. 5가지 동적 반영 요소(계절·검증된 카피·검증된 이미지·고객 피드백·순서) + Pain Point 표현 4기법 + 법적 회피 운용.

## 핵심 1 — 상세페이지 동적 운영 5요소

> 한 번 만들고 끝 X. 광고·외부·고객 데이터로 매주 갱신.

| 요소 | 운영 방법 |
|---|---|
| **계절·시즌** | 봄/여름/가을/겨울 + 명절·신학기 등 사회적 흐름 반영 |
| **광고에서 검증된 카피** | CTR/ROAS 우수 카피 → 상페 최상단 적극 활용 (법적 문제 없으면) |
| **광고에서 검증된 이미지** | 광고 CTR 테스트 우수 이미지 → 상페 대표 이미지로 재활용 |
| **고객 직접 피드백** | 쿠팡·스마트스토어 리뷰 지속 모니터링. 별점 1점 급증 시 즉시 원인 파악·수정 |
| **순서 유동성** | 데이터로 입증되면 화자/블록 순서 과감히 재조정 (연예인 상단 vs 리뷰 상단 등) |

필수 요소: **고통지수 표기**, **이미지·카피 일치** (= 직관적 이해도 향상).

## 핵심 2 — Pain Point 표현 4기법

| 기법 | 정의 |
|---|---|
| **1. 고객 행동 패턴 + 직접 명시** | 고객 행동 이해 후 문제 직접 명시 |
| **2. 제품 성능 → 직접 해결** | 제품 기능을 문제 해결과 직결 |
| **3. 전문가/고객 시선 → 간접 해결** | 제3자 화자로 간접 제시 |
| **4. Storytelling 공감** | 실제 사용 동기 사례로 공감 유도 |

→ [[상페 구조]] §USP+Pain+손실 3축 보강 자료.

## 핵심 3 — 법적 회피·결과 표현 (실전 운용)

> 식약처·전자상거래법·과대광고 회피하면서 효과 암시.

- 피부과 이용 "효과" 직설 X → **"동창회 가니 성형수술했냐고 묻더라"** 같은 상황 묘사
- 법적 문제 우려 시 → 일단 만들고 테스트 후 신고 들어오면 식약처 시정명령에 맞춰 수정 (완벽 사전검증은 비실용)

→ 환사장 명제 "대중은 똑똑하지 않음 + 그럴싸해보이면 구매" + [[프레임 분리]] (식약처 인증 연고 사례)와 같은 결.
